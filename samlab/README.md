# 🔧 מבריק לאב · SamLab

תוכנת מעבדה ל‑**Windows 11** לתיקון והחייאת מכשירי **Samsung** — חלופה פתוחה, נקייה ובעברית ל‑SamFwTool, לזרימת העבודה הלגיטימית של מעבדה.

> **מה זה עושה:** הורדת קושחה רשמית, פלאש (unbrick) ומידע/מעבר מצבים.
> **מה זה *לא* עושה, במכוון:** עקיפת FRP או הסרת חשבון Google/Samsung — ראה סעיף "מה לא נכלל ולמה".

---

## הרעיון: תזמור מנועים בדוקים, לא המצאה מחדש

פרוטוקול ה‑FUS (הורדת קושחה) ופרוטוקול Odin (פלאש) הם רגישים. מימוש שלהם מאפס ולא‑בדוק = כלי שלא עובד, ובמקרה של פלאש — כלי שעלול להפוך טלפון של לקוח ל‑brick. לכן SamLab הוא **ממשק מודרני בעברית מעל מנועים פתוחים ובדוקים**:

| רכיב | מנוע | למה |
|------|------|-----|
| הורדת קושחה רשמית | [`samloader`](https://github.com/nlscc/samloader) | ה‑crypto הנכון של שרתי Samsung FUS, כולל פענוח `.enc4/.enc2` |
| פלאש / זיהוי / PIT | [Heimdall](https://glassechidna.com.au/heimdall/) | מנוע Odin/LOKE פתוח, בטוח וחוצה‑פלטפורמות |
| מידע ומעבר מצבים | ADB (platform-tools) | קריאת מידע ואתחול אל Download Mode |

היתרון על SamFwTool: פתוח, ניתן לביקורת, מאחד הורדה+פלאש בזרימה אחת, בלי bloatware — ובלי סיכון של פלאש שביר מהזיכרון.

## התקנה ב‑Windows 11

### 1. Python
התקן **Python 3.10 ומעלה** מ‑[python.org](https://www.python.org/downloads/windows/) — סמן **"Add Python to PATH"** בהתקנה. (Tkinter מגיע מובנה, אין צורך בהתקנה נפרדת.)

### 2. הכלים החיצוניים
- **Heimdall:** הורד והנח את `heimdall.exe` בתיקיית `tools/`.
- **ADB:** הורד את [platform-tools](https://developer.android.com/tools/releases/platform-tools), והעתק את `adb.exe` (עם `AdbWinApi.dll`, `AdbWinUsbApi.dll`) לתיקיית `tools/`.
- **samloader:** מותקן אוטומטית דרך `pip` (ראה מטה).

### 3. דרייבר USB (קריטי לפלאש)
כדי ש‑Heimdall יראה את המכשיר ב‑Download Mode צריך דרייבר **libusb / WinUSB** לממשק ה‑Download:
1. התקן את **Samsung USB Driver for Mobile Phones** (הרשמי של Samsung).
2. אם Heimdall עדיין לא מזהה — הכנס את המכשיר ל‑Download Mode, הרץ **[Zadig](https://zadig.akeo.ie/)**, בחר את התקן ה‑Samsung, והתקן לו דרייבר **WinUSB** / **libusbK**.

### 4. בנייה או הרצה
```bat
:: הרצה מהירה (מפתחים)
run.bat

:: בניית קובץ EXE יחיד להפצה במעבדה
build.bat        ::  → dist\SamLab.exe
```
לאחר הבנייה, העתק את תיקיית `tools/` לצד `SamLab.exe`.

## שימוש

### ⬇ טאב "הורדת קושחה"
1. בחר **דגם** ו‑**אזור/CSC** מהרשימות הנפתחות (ישראל בראש: `ILO`/`CEL`/`PCL`/`PTR`, ועוד ~80 אזורים), או הקלד כל קוד ידנית. אפשר לציין **גרסה מסוימת** או להשאיר ריק לעדכנית.
2. "בדוק גרסה עדכנית" → מציג את הגרסה האחרונה בשרתי Samsung.
3. "הורד + פענח" → מוריד ומפענח אוטומטית ל‑`.tar.md5` מוכנים לפלאש.

### 💾 טאב "פלאש"
1. הכנס את המכשיר ל‑**Download Mode** (בכיבוי: `Vol‑` + `Vol+` בזמן חיבור הכבל, ואשר עם `Vol+`).
2. "זהה מכשיר" → מוודא ש‑Heimdall רואה את הטלפון.
3. "קרא PIT" → קורא את טבלת המחיצות (נדרש למיפוי אוטומטי).
4. בחר את קובצי ה‑`.tar.md5`: **BL / AP / CP / CSC** (ו‑USERDATA לרשות).
5. "הכן תוכנית פלאש" → מחלץ, מפרק `lz4` וממפה כל image למחיצה לפי ה‑PIT. התוכנית מוצגת ביומן.
6. "בצע פלאש" → אישור, ואז כתיבה. **אל תנתק את הכבל בזמן הפלאש.**

### 🩺 טאב "מכשיר ומצבים"
למכשיר דלוק עם ניפוי‑USB: קריאת מידע ואתחול אל Download / Recovery / Bootloader / רגיל.

### ⚙ טאב "הגדרות"
נתיבי הכלים (או השאר ריק לאיתור אוטומטי מ‑`tools/` ומ‑PATH), ו"בדוק זמינות כלים".

### 🧪 טאב "מנוע ישיר" (ניסיוני)
מימוש עצמאי של פרוטוקול **Odin/LOKE** מעל USB (בלי Heimdall), שנכתב לפי קוד המקור של Heimdall. **בשלב זה מיושם רק הנתיב הבטוח שאינו כותב דבר למכשיר:**
- **חבר + פתח סשן** — `handshake` (ODIN→LOKE) ופתיחת סשן.
- **קרא PIT (בטוח)** — מוריד ומפרק את טבלת המחיצות ומציג `ID | מחיצה | קובץ פלאש`, עם אפשרות לשמור `.pit`.
- **סיים סשן + אתחל** — סוגר סשן ומאתחל.

אין כאן פעולת פלאש (כתיבה) — **אפס סיכון brick**. המטרה: לאמת שהפרוטוקול עובד על מכשיר בר‑הקרבה (למשל Galaxy A16) לפני שנרחיב לכתיבה.

**דרישות למנוע הישיר:**
- `pip install pyusb` (כלול ב‑requirements) **וגם** ספריית **libusb‑1.0**:
  - Windows: הנח את `libusb-1.0.dll` לצד התוכנה, או התקן libusb.
  - הכנס את המכשיר ל‑Download Mode, הרץ **[Zadig](https://zadig.akeo.ie/)**, בחר את התקן ה‑Samsung והתקן לו דרייבר **libusbK** (או WinUSB).
- אם "חבר" נתקע ללא תשובה — סמן את **"שלח ZLP אחרי כל פקודה"** ונסה שוב.

> הערה ל‑A16 (MediaTek): פרוטוקול ה‑LOKE הבסיסי זהה, אך אם הזיהוי או קריאת ה‑PIT נכשלים, דווח לי את הפלט מהיומן ונכוונן.

## מה לא נכלל, ולמה

SamLab **אינו** כולל עקיפת **FRP** (Factory Reset Protection) או הסרת **חשבון Google / Samsung / Knox**. אלה מנגנוני הגנה מפני גניבה, והשימוש העיקרי של כלי‑עקיפה "משופרים" בעולם הוא הכשרת מכשירים גנובים. ללקוח עם בעלות אמיתית ששכח סיסמה, המסלול הנכון והחוקי הוא:
- שחזור חשבון Google / Samsung הרשמי, או
- פנייה ל‑Samsung / ליבואן עם **הוכחת רכישה** להסרת הנעילה.

## מגבלות ידועות

- **Heimdall ומכשירים חדשים:** בחלק מדגמי Samsung החדשים (2021+) Heimdall עלול לדרוש פורק מעודכן. עדכן ל‑fork נתמך אם הזיהוי נכשל.
- **שינויים בשרתי FUS:** שרתי Samsung משתנים מדי פעם; אם ההורדה נכשלת, עדכן את `samloader` (`pip install -U samloader`) או השתמש בכלי הורדה חלופי.
- **פלאש = אחריות המשתמש:** עבוד תמיד עם קושחה מקורית התואמת בדיוק לדגם ולאזור.
