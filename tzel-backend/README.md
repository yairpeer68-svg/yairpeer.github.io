# צל דיגיטלי — הפעלת השרת 🕵️

האפליקציה ב-`/tzel/` עובדת כרגע במצב הדגמה (דוח לדוגמה). המדריך מסביר איך מדליקים בדיקה אמיתית מול מאגרי דליפות. **זמן: ~20 דקות. עלות: ₪0 להתחלה.**

## עקרון מנחה (חשוב)

צל דיגיטלי הוא כלי **הגנה עצמית** — הוא בודק רק את הפרטים של המשתמש עצמו. זה מה שהופך אותו למוצר לגיטימי לחנות ולא לכלי מעקב. שמור על העיקרון הזה בכל שינוי עתידי.

## שלב 1: פרויקט Supabase (חינם)

1. https://supabase.com ← **Start your project** (התחברות עם GitHub)
2. **New project** ← שם `tzel` ← סיסמת DB (שמור) ← Region אירופה ← **Create**

## שלב 2: מסד הנתונים

**SQL Editor ← New query** ← הדבק את כל `schema.sql` ← **Run**.

## שלב 3: כניסה אנונימית

**Authentication ← Sign In / Up** ← הפעל **Anonymous Sign-Ins** ← Save.

## שלב 4: מפתח בדיקת דליפות (אופציונלי אך מומלץ)

1. https://haveibeenpwned.com/API/Key ← רכוש מפתח API (~$4/חודש, בדיקות בלתי מוגבלות כמעט)
2. שמור את המפתח לשלב הבא
   *(בלי המפתח האפליקציה עדיין עובדת ומחזירה בדיקות הגנה כלליות, רק בלי בדיקת הדליפות בפועל.)*

## שלב 5: פריסת פונקציית הסריקה

1. **Edge Functions ← Deploy a new function ← Via Editor**
2. שם: `scan`
3. הדבק את כל `functions/scan/index.ts` ← **Deploy**
4. **Edge Functions ← Secrets** ← הוסף (אם רכשת מפתח): `HIBP_API_KEY` = המפתח משלב 4

## שלב 6: חיבור האפליקציה

1. **Project Settings ← API** ← העתק **Project URL** ו-**anon public key**
2. ב-`tzel/index.html`, מלא למעלה:
   ```js
   const CONFIG = { SUPABASE_URL: "…", SUPABASE_ANON_KEY: "…" };
   ```
   (או שלח לי את שני הערכים ואמלא בשבילך.) דחוף ל-main.
3. התג למעלה יתחלף ל"מחובר ●", והסריקה תרוץ מול מאגרים אמיתיים.

## המודל העסקי

- **חינם:** סריקה + הממצא הקריטי הראשון גלוי (מספיק כדי שהמשתמש יבין שהוא חשוף)
- **פרימיום ₪19.90/חודש:** כל הממצאים, כל צעדי ההגנה, ו**ניטור מתמשך** שמתריע על דליפה חדשה
- ניטור מתמשך = הסיבה שהמנוי מתחדש כל חודש (הערך לא נגמר)
